# TBD Reforger Platform - Master Architecture & Context Document

**Purpose:** This document contains the complete context, UI blueprints, and system architecture for the "TBD Reforger Event Platform." It is designed to be fed into an LLM (like Claude) to provide 100% project context for generating backend code, data models, and integration scripts.

---

## 1. Project Overview
We are building a premium, all-in-one web suite for a tactical Arma Reforger Milsim community named "TBD". The platform handles user authentication, event scheduling, mission creation (via a visual 2D web editor), server telemetry, and community doctrine.

**Tech Stack:**
- **Frontend**: React (Dark theme, Tailwind/CSS, highly visual and interactive).
- **Backend**: Go (RESTful API, JWT handling).
- **Database**: TBD (Likely PostgreSQL for relational data, plus JSON document storage for missions).
- **Game Engine**: Arma Reforger (Enfusion Engine).

---

## 2. Core Integrations Required

### A. Discord Integration
- **OAuth2 Login**: The platform uses Discord for primary authentication. No local passwords.
- **Role Syncing**: The Go backend must sync Discord roles (e.g., @Admin, @MissionMaker) to web permissions.
- **Webhooks**: The CMS must push news and event announcements directly to the Discord `#announcements` channel via webhook embeds.

### B. Arma Reforger Server Integration
- **Identity Linking**: A system linking a Discord ID to an Arma Enfusion ID (using an existing Go route `POST /api/me/link`).
- **Telemetry Ingestion**: The backend must receive live server data (Server FPS, Uptime, Player Count) and post-match data (Kills, Deaths, Vehicles Destroyed, Command Win Rate) to populate the Global Leaderboards.
- **RCON / Command Line**: Basic server controls (Restart, Change Map) integrated into the Admin panel.
- **Mission Injection**: The 2D Mission Editor on the web must export a strict `mission.json` file and push it to the server's mission directory.

## 3. Data Models (High-Level)

### Users & Authentication
- `discord_id` (Primary Key)
- `arma_id` (Linked Enfusion ID)
- `username` / `avatar_url`
- `role` (Admin, Mission Maker, Enlisted)
- `attendance_rate`
- `total_deployments`

### Events & Deployments (Operations)
- `event_id`
- `mission_id` (Foreign Key to Mission Library)
- `start_time` / `status` (Scheduled, Live, Completed)
- `orbat_slots` (JSON structure mapping squads like "Alpha 1-1" to specific user IDs).

### Leaderboard Stats (Telemetry)
- `user_id`
- `kills` / `deaths` / `longest_kill`
- `team_kills` (Wall of Shame)
- `vehicles_destroyed`
- `command_wins` (If playing Platoon HQ)

### Mission Library
- `mission_id`
- `author_id`
- `title`, `terrain` (Everon/Arland), `weather`, `time_of_day`
- `game_mode` (PvP / PvE)
- `max_players`
- `status` (Pending Approval, Live)
- `json_payload` (The massive object detailing spawn points, vehicles, and map markers generated by the 2D Editor).

## 4. Frontend Architecture & Exhaustive UI Blueprints

The frontend is divided into specific, highly-polished pages. Below is the EXACT layout, fields, and grid structure required for every single page. The backend Go API must be designed to fetch and serve all the specific data points listed in these blueprints.

### 1. Server Intel
*Real-time server status without revealing tactical advantages (Anti-Ghosting).*
- **Layout**: A clean 3-column dashboard grid.
- **Column 1 (Server Health)**: A large card showing the Server Name, Uptime (e.g., "Live for 02:14:00"), Player Count (e.g., "55/64"), and Server FPS/Tickrate (Arma players love knowing if the server is healthy).
- **Column 2 (Current Operation)**: A card showing a static thumbnail of the current terrain, the name of the active mission, and in-game weather/time (e.g., "Everon - 14:00 Local - Overcast").
- **Column 3 (Connection)**: A card with the Direct Connect IP, Port, required Modpack version, and a massive `[Launch via Steam / Join Server]` button.

### 2. Announcements (News Feed)
*Community updates and patch notes.*
- **Layout**: A centered, constrained width column (like a blog).
- **Content**: A vertical stack of large "News Cards".
- **Card Design**: Each card has a thumbnail image on the left, and on the right: a Date Pill, a bold Title, Author Name, a snippet of text, and a `[Read Full Briefing]` button.

### 3. My Deployments
*The player's personal service record.*
- **Layout**: Split horizontal sections.
- **Top Section (Upcoming)**: A grid of "Deployment Cards". Each card shows the Operation Name, a countdown timer, and a highly visible badge showing their **Assigned ORBAT Slot** (e.g., `US Army - Alpha 1-1 - Medic`).
- **Bottom Section (Service History)**: A data table listing past operations they attended, their role, and a link to the AAR.

### 4. Global Leaderboards
*Competitive community stats pulled from server telemetry.*
- **Layout**: A classic leaderboard layout with a search bar to find specific players.
- **Header**: Tabs to switch between categories: `K/D Ratio`, `Command Win Rate`, `Missions Played`, and `Team Kills (Wall of Shame)`.
- **The Podium (Top)**: Three visually distinct cards for 1st, 2nd, and 3rd place, with larger avatars and gold/silver/bronze borders.
- **The Ranks (Bottom)**: A clean list/table for ranks 4 through 50.

### 5. Mission Library
*The database of all custom missions.*
- **Header**: A search bar and filter dropdowns (`Terrain`, `Faction`, `Player Count`).
- **Layout**: A responsive CSS Grid of "Mission Cards" (3 or 4 per row).
- **Card Design**: A terrain thumbnail, the Mission Title, Author Avatar, a pill indicating "My Mission" vs "Global", and a `[View Overview]` button.

### 6. SOPs & Manuals (The Wiki)
*The community knowledge base and anti-retardation guide.*
- **Layout**: 2-column layout. Left sidebar (25% width) for Navigation, Right area (75% width) for Content.
- **Left Navigation**: A vertical menu of document categories:
  - `Rules of Engagement (ROE)`
  - `Radio Protocol (ACRE/TFAR)`
  - `Vehicle Database (Strengths/Weaknesses)`
  - `Friend or Foe (IFF) Recognition`
  - `Medical Flowchart (ACE)`
- **Right Content Area**: A clean, highly readable Markdown text area for the active document, featuring large headings, tables (e.g., Vehicle armor values, amphibious capabilities), and warning callout boxes for common mistakes.

### 7. Modpacks
*Dependencies and downloads.*
- **Layout**: A simple grid of wide cards.
- **Card Design**: E.g., "Core Modern Modpack". Shows the total file size, a list of the major mods included, and a massive `[Connect to Server to Download]` or `[Open in Reforger Workshop]` button.

### 8. Event Manager
*For scheduling Tuesday events.*
- **Layout**: Split screen.
- **Left Side**: A large calendar UI. Clicking a day opens the form on the right.
- **Right Side**: A form to "Schedule Operation". Inputs: Date, Time, Select Mission (dropdown from library), and `[Publish Event]` button.

### 9. Mission Approvals
*Quality control for community missions.*
- **Layout**: A data table.
- **Columns**: `Submitted By`, `Mission Name`, `Terrain`, `Date`.
- **Action Column**: A group of three buttons: `[Review JSON]`, `[Approve for Live]`, `[Reject]`.

### 10. Personnel Roster
*User management.*
- **Layout**: A data table with a heavy search bar at the top.
- **Columns**: `Discord Name`, `Arma ID (Linked)`, `Rank/Role`, `Warnings`.
- **Action Column**: `[Edit User]`, `[Ban]`.

### 11. Content Manager
*The centralized CMS for publishing News and updating the SOP Wiki.*
- **Header**: Tabs to switch between `[Create Announcement]` and `[Edit Wiki Page]`.
- **Layout (Announcement Mode)**:
  - **Meta Inputs**: `Title`, `Category Tag` (e.g., Update, Event), and a `Thumbnail Image Upload` drag-and-drop zone.
  - **Editor**: A massive Rich Text Editor (WYSIWYG) with formatting toolbars.
  - **Publishing Options**: A toggle switch for `Push to Discord Webhook` (to automatically ping the Discord server).
  - **Action**: `[Publish Announcement]`.

### 12. Audit Logs
*Admin papertrail.*
- **Layout**: A full-width, terminal-style view. Dark background, monospace text.
- **Content**: A running list of system events: `[14:22:01] Admin Dave approved mission 'Op Overcast'`.
